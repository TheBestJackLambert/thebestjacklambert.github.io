// THE_MATH

// DFT EQUATION

X[k] = Σ x[n] · e^(-j·2π·k·n/N)

     = Σ x[n] · [cos(2πkn/N) - j·sin(2πkn/N)]

for k = 0, 1, ..., N-1

The magnitude |X[k]| tells us how much of frequency k is present. Try adding a 5 Hz sine in the interactive above — you'll see a single spike at bin 5.

// INVERSE FOURIER TRANSFORM

x[n] = (1/N) Σ |X[k]| · cos(2πkn/N + φ[k])

for n = 0, 1, ..., N-1

Reconstructs a time-domain signal from its frequency components — undoing what the DFT computed.

// SPECTRAL DENOISING PIPELINE

1. mag, phase = FFT(noisy_signal)
2. threshold = mean(mag) × 7
3. zero bins where mag[k] < threshold
4. subtract noise floor from remaining bins
5. clean_signal = IFT(mag, phase)[:N]

Weak frequency bins are typically noise. Strong ones are signal. The threshold separates them.

// ARCHITECTURE

The library has a clean two-file separation: trig.py implements low-level math primitives (sin, cos, arctan) from Taylor series, and signals.py builds high-level DSP applications on top. Everything is pure Python arithmetic — no NumPy, no SciPy, no math module. The JavaScript port mirrors this 1:1.
